🔐 FUNCIONES DISPONIBLES:

DECODIFICACIÓN:
├─ decodeToken(token)           → Decodifica JWT
└─ getToken()                   → Obtiene JWT del storage

INFORMACIÓN DEL USUARIO:
├─ getUserInfo()                → Todos los datos
├─ getUserRole()                → Solo el rol actual
├─ getUserRoles()               → Array de roles
└─ getFullUserData()            → Idem getUserInfo

VERIFICACIÓN DE ROLES:
├─ hasRole('admin')             → ¿Tiene este rol?
├─ hasAnyRole(['admin', 'user']) → ¿Tiene alguno?
└─ hasAllRoles(['admin', 'edit']) → ¿Tiene todos?

VALIDACIÓN:
├─ isTokenValid()               → ¿Token válido?
└─ getTokenExpirationTime()     → Minutos hasta expiración

LIMPIEZA:
└─ clearToken()                 → Limpia todo (logout)

COMPOSABLE REACTIVO:
├─ useAuth()                    → Accede a todo con reactividad

❓ PREGUNTAS FRECUENTES:

P: ¿Dónde guardar el token? ¿localStorage o sessionStorage?
R: sessionStorage es más seguro (se borra al cerrar pestaña)
   localStorage persiste pero es menos seguro

P: ¿Puedo usar múltiples roles?
R: Sí, el backend puede retornar un array en "Roles"
   O usar hasAnyRole(['admin', 'editor'])

P: ¿Qué pasa si el token expira?
R: isTokenValid() retorna false y el interceptor lo detecta
   Se limpia automáticamente y redirige a login

P: ¿Puedo proteger rutas con roles?
R: Sí, agrega meta: { requiresRole: 'admin' } en la ruta
   Y valida en router.beforeEach()

P: ¿Cómo cierro sesión?
R: Usa logout() del composable o clearToken()
   Eso limpia todo de sessionStorage y localStorage
